# 3.8.1 Basic Pagination

| Section | Details |
| :--- | :--- |
| **Focus & Purpose** | Breaking a large dataset (array or collection) into multiple chunks to display across a series of pages (e.g., Blog Page 1, Page 2). |
| **Key Concepts / Details** | - **Key**: `pagination:`.<br>- **Data Source**: `data: collections.posts` points to the array to split.<br>- **Size**: `size: 10` determines items per page.<br>- **Current Page Data**: The template renders *for the current page's chunk*.<br>- **Output**: Generates `page/0/index.html` (or similar) for each chunk. |
| **One-Line Recall** | The mechanism for splitting a single array of data into multiple rendered output pages. |
