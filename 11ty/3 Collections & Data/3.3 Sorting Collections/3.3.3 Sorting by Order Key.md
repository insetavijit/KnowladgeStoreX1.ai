# 3.3.3 Sorting by Order Key

| Section | Details |
| :--- | :--- |
| **Focus & Purpose** | Implementing manual sort control by adding a custom numerical property (like `order` or `weight`) to front matter for precise positioning. |
| **Key Concepts / Details** | - **Front Matter**: Adding `order: 1`, `order: 2` etc.<br>- **Sort Function**: `collection.sort((a, b) => a.data.order - b.data.order)`.<br>- **Navigation**: Critical for sidebars or step-by-step tutorials.<br>- **Defaults**: Handling items that miss the sort key (treat as 0 or Infinity).<br>- **Zero-Padding**: Not required for numeric sort, but helpful for file naming. |
| **One-Line Recall** | Manually defining sequence using a custom numeric field in front matter. |
