# 3.6.2 Defaulting Front Matter

| Section | Details |
| :--- | :--- |
| **Focus & Purpose** | Using directory data to set default configuration values like `layout`, `tags`, or `permalink` for an entire section of the site. |
| **Key Concepts / Details** | - **Default Layout**: `{"layout": "post.njk"}` in `posts.json`.<br>- **Default Tags**: `{"tags": ["blog", "tech"]}` applies to all.<br>- **Permalink Structure**: Setting a common URL pattern for the folder.<br>- **Overrides**: Individual files can still set a different layout to opt-out.<br>- **Dry Principle**: drastically reduces repetition in individual file headers. |
| **One-Line Recall** | Setting common front matter keys (layout, tags) once for all files in a directory. |
