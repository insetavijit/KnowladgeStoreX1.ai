# 3.5.5 Data Preprocessing

| Section | Details |
| :--- | :--- |
| **Focus & Purpose** | Cleaning, filtering, or formatting raw data fetched from an API before it is passed to the templates, ensuring clean logic-less views. |
| **Key Concepts / Details** | - **ETL Pipeline**: Extract (fetch), Transform (process), Load (return).<br>- **Caching**: Using `@11ty/eleventy-fetch` to cache API responses locally.<br>- **Optimization**: Reducing payload size by discarding unused fields.<br>- **Normalization**: Converting inconsistent API data into a standard internal format.<br>- **Reliability**: ensuring fallback data if the API returns a partial response. |
| **One-Line Recall** | The practice of transforming and caching raw external data into a template-friendly format. |
