# 3.5.2 JSON & YAML Data

| Section | Details |
| :--- | :--- |
| **Focus & Purpose** | Using static data files (JSON/YAML) in the `_data` directory to store site-wide constants, configuration, or content lists. |
| **Key Concepts / Details** | - **File Types**: `.json` and `.yaml` (requires `eleventy-plugin-yaml` in some versions, pure yaml allowed often).<br>- **JSON Syntax**: Standard JSON format (key-value pairs).<br>- **Use Cases**: Site metadata (title, author), navigation menus, footer links.<br>- **Simplicity**: No logic, just data storage.<br>- **Parsing**: Automatically parsed and exposed as JS objects. |
| **One-Line Recall** | Storing static global data in standard JSON or YAML text formats. |
