# 3.5.3 JavaScript Data Files

| Section | Details |
| :--- | :--- |
| **Focus & Purpose** | Using `.js` files in `_data` to execute logic, access environment variables, or return dynamic data structures. |
| **Key Concepts / Details** | - **CommonJS**: `module.exports = { ... }`.<br>- **Dynamic Values**: `date: new Date()` executed at build time.<br>- **Environment**: Accessing `process.env.API_KEY` securely.<br>- **Functions**: Can export a function that returns data.<br>- **Flexibility**: Allows computing data rather than just storing it statically. |
| **One-Line Recall** | Executing JavaScript logic to generate global data objects at build time. |
