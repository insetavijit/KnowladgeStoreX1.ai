## **Display Rules — Ruleset**

### **Question**

- Start with H4-style heading:  
    `#### **N. Question?**`
    

---

### **Sections (Fixed Order — H5 Style)**

1. **Interview-Ready Answer**
    
2. **Concept & Clarification**
    
3. **Agent-Centric Code Example**
    

---

### **Headings**

- Each section uses **H5-style bold headings**.
    

---

### **Interview-Ready Answer**

- One concise, professional paragraph first.
    
- Must sound like a **senior engineer**.
    
- Include:
    
    - What it is
        
    - Why it matters
        
    - How it’s built (high level)
        
    - When to use vs not
        
- Optionally include a one-line **Counter:** trade-off.
    

---

### **Concept Section Structure**

Start with a brief intro, then use **bold sub-labels**:

- **What it is**
    
- **Why it exists**
    
- **How it works**
    
- **Agent components**
    
- **Defaults & behaviors**
    
- **Edge cases & pitfalls**
    
- **Utilities / variations**
    

Use bullets where helpful.

---

### **Code Example**

- Exactly **one fenced block**.
    
- Must show a **real agent pattern**, such as:
    
    - ReAct loop
        
    - Tool calling
        
    - Planner–executor
        
    - LangGraph / CrewAI / AutoGen style flow
        
- Follow with **Explanation:** paragraph.
    

---

### **Styling**

- **Bold** → headings, labels
    
- _Italic_ → emphasis
    
- Inline code → APIs, classes, tokens, patterns
    

---

### **Layout & Consistency**

- Blank lines between blocks.
    
- No dense text.
    
- Same order, tone, and labels.
    
- No meta commentary.
    

---

## **Agent Content Framework**

### **Interview-Ready Answer**

- Lead with a system-level summary.
    
- Emphasize autonomy, control loop, tools, memory, and trade-offs.
    

---

### **Concept & Clarification**

Explain from zero knowledge, covering:

- Agent vs single LLM call
    
- Control loop & autonomy
    
- Tools and environment interaction
    
- Memory (short/long-term)
    
- Planning & reflection
    
- Failure modes & safety
    
- Single vs multi-agent trade-offs
    

---

### **Agent-Centric Code Example**

- One minimal, relevant example focused on:
    
    - Tool orchestration
        
    - State passing
        
    - Loop / graph
        
    - Explicit agent behavior
        

---
