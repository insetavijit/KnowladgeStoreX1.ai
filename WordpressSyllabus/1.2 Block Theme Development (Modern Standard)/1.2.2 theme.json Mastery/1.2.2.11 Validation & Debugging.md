| **Section**                            | **Interview Learning Goals**                                                   | **Key Technical Details to Study**                                                                          |
| -------------------------------------- | ------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------- |
| **Validation & Debugging**             | Explain how to ensure theme.json correctness and troubleshoot issues.          | JSON validation, error logs, editor warnings, dev tools; debugging workflow.                                |
| **[[JSON Syntax Validation]]**         | Ensure valid JSON syntax.                                                      | JSON linters, IDE validation, online validators; common syntax errors (trailing commas, quotes).            |
| **[[Schema Validation]]**              | Validate against the official theme.json schema.                               | Using `$schema` URL; IDE intellisense; schema validation tools; catching structural errors.                 |
| **[[WordPress Debug Mode]]**           | Enable debugging to catch theme.json errors.                                   | `WP_DEBUG`, `WP_DEBUG_LOG`, `WP_DEBUG_DISPLAY`; reading debug.log; error messages.                         |
| **[[Browser DevTools]]**               | Use browser tools to inspect generated CSS.                                    | Inspecting CSS custom properties; checking applied styles; debugging specificity issues.                    |
| **[[Editor Warnings]]**                | Recognize and resolve editor warnings.                                         | Site Editor error messages; block validation warnings; style application issues.                            |
| **[[Common Errors]]**                  | Identify typical theme.json mistakes.                                          | Invalid slugs, wrong data types, missing required keys, incorrect nesting; typos.                           |
| **[[CSS Variable Inspection]]**        | Verify CSS custom properties are generated correctly.                          | Checking `:root` styles; verifying preset variables; troubleshooting missing variables.                     |
| **[[Testing Workflow]]**               | Establish a systematic testing approach.                                       | Local development, staging environment, cross-browser testing; version compatibility testing.               |
| **[[Validation Tools]]**               | Use available tools for validation.                                            | Theme Check plugin, JSON validators, schema validators; automated testing.                                  |
| **[[Performance Profiling]]**          | Check theme.json impact on performance.                                        | CSS output size, render performance, loading time; optimization opportunities.                              |
