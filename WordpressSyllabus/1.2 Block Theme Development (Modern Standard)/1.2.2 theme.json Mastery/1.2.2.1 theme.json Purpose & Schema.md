| **Section**                     | **Interview Learning Goals**                                                | **Key Technical Details to Study**                                                                            |
| ------------------------------- | --------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **theme.json Overview**         | Explain the central role of theme.json in modern WordPress themes.          | Replaces add_theme_support; centralizes checking; controls editor UI; generates CSS variables & styles.       |
| **[[Purpose of theme.json]]**   | Define why theme.json exists and what problems it solves.                   | Unification of editor and front-end styles; reduction of PHP/CSS code; standardized configuration API.        |
| **[[JSON Structure & Schema]]** | Describe the root-level keys and structure of the file.                     | `$schema`, `version` (v2/v3), `settings`, `styles`, `customTemplates`, `templateParts`, `patterns`.           |
| **[[Settings vs Styles]]**      | Differentiate between 'settings' (configuration) and 'styles' (appearance). | Settings = what user CAN do (UI controls, palettes); Styles = what site LOOKS like (defaults, CSS output).    |
| **[[Global Configuration]]**    | Explain how to enable/disable specific editor features.                     | `appearanceTools`, `layout` (content/wide), `border`, `color`, `typography`, `spacing` controls.              |
| **[[Presets Definition]]**      | Show how to define design tokens (variables).                               | Defining `palette`, `gradients`, `fontSizes`, `fontFamilies`, `spacingScale`; creating CSS Custom Properties. |
| **[[Global Styles Root]]**      | Explain how to apply styles to the `<body>` (root).                         | `styles.typography`, `styles.color.background`, `styles.color.text`; matches `body {}` selector logic.        |
| **[[Elements Styling]]**        | Describe styling generic HTML elements globally.                            | `styles.elements.h1-h6`, `link`, `button`, `caption`, `cite`; targeting specific HTML tags via JSON.          |
| **[[Block-Specific Config]]**   | Explain how to target specific blocks within theme.json.                    | `settings.blocks.core/paragraph`; `styles.blocks.core/button`; scoped settings vs scoped styles.              |
| **[[Schema Validation]]**       | Demonstrate how to ensure the JSON file is valid and auto-completable.      | Using `$schema` URL ( ); IDE intellisense setup; avoiding syntax errors.                                      |
| **[[Cascade & Specificity]]**   | Explain how theme.json rules merge with user and core styles.               | Core defaults -> Theme (theme.json) -> User (Global Styles UI); CSS Specificity implications.                 |
| **[[Version Difference]]**      | Briefly touch on version 2 vs version 3 changes (if relevant).              | v2 (standard) vs v3 (future/trunk); spacing spacing changes; element differences; why version key matters.    |
