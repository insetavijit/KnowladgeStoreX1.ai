| **Section**                            | **Interview Learning Goals**                                                   | **Key Technical Details to Study**                                                                          |
| -------------------------------------- | ------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------- |
| **Color Palettes & Gradients**         | Explain how to create and manage brand color systems.                          | Custom palettes, gradients, duotone filters; disabling default colors; CSS custom properties generation.    |
| **[[Custom Color Palettes]]**          | Define custom color palettes in theme.json.                                    | `settings.color.palette` array; `name`, `slug`, `color` properties; generates `--wp--preset--color--slug`.  |
| **[[Disabling Default Palettes]]**     | Remove WordPress default colors for brand consistency.                         | `settings.color.defaultPalette: false`; enforcing strict brand colors only.                                  |
| **[[Gradient Presets]]**               | Create custom gradient options.                                                | `settings.color.gradients` array; linear/radial gradients; `--wp--preset--gradient--slug` variables.        |
| **[[Duotone Filters]]**                | Apply duotone color effects to images.                                         | `settings.color.duotone` array; two-color filters; image block integration; accessibility considerations.   |
| **[[Color Slugs & Naming]]**           | Best practices for naming color tokens.                                        | Semantic names (primary, secondary) vs descriptive (ocean-blue); consistency; maintainability.              |
| **[[CSS Custom Properties]]**          | Understand how colors become CSS variables.                                    | Auto-generated `--wp--preset--color--*` variables; usage in custom CSS; cascade and inheritance.            |
| **[[Link Color Control]]**             | Manage link color settings separately.                                         | `settings.color.link: true`; separate link color picker; default vs hover states.                           |
| **[[Background & Text Colors]]**       | Control background and text color options.                                     | `settings.color.background`, `settings.color.text`; enabling/disabling specific color controls.             |
| **[[Color in Block Supports]]**        | Understand block-level color support.                                          | How blocks inherit color settings; block-specific color overrides; editor vs front-end rendering.           |
| **[[Accessibility Considerations]]**   | Ensure color choices meet accessibility standards.                             | WCAG contrast ratios; color blindness; providing sufficient color contrast; testing tools.                  |
