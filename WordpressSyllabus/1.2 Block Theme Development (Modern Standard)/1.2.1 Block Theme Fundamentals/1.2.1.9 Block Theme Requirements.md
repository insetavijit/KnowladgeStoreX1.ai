# 1.2.1.9 Block Theme Requirements

**Overview**
To officially be recognized as a **Block Theme** by WordPress (and the repository), a theme must meet specific technical criteria. These requirements define the minimum viable product (MVP) for FSE. Understanding these ensures your theme is compliant, future-proof, and capable of utilizing the full suite of Site Editor features.

| **Section**                                 | **Interview Learning Goals**                                                    | **Key Technical Details to Study**                                                                             |
| ------------------------------------------- | ------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| **Mandatory Files**                         | What *must* exist?                                                              | `style.css` (header), `templates/index.html`, `theme.json`, `index.php` (empty fallback).                      |
| **[[HTML Comments Markup]]**                | Standards for block validity.                                                   | Valid JSON attributes in comments; strictly closed tags; no PHP tags inside HTML templates.                     |
| **[[Required Headers]]**                    | Metadata in style.css.                                                          | "Requires at least: 5.9"; "Tested up to"; standard theme tags (e.g., `block-theme`, `full-site-editing`).      |
| **[[Accessibility Requirements]]**          | Accessibility Ready standards for blocks.                                       | Skip links (handled by core); proper heading hierarchy; keyboard navigation; color contrast in default palette. |
| **[[Review Guidelines]]**                   | Passing the Theme Review Team (TRT) check.                                      | No hardcoded strings (i18n); licensing; no plugin-territory functionality; escaping output (if PHP is used).   |
| **[[Screenshot Standards]]**                | Visual representation.                                                          | 1200x900px (4:3 ratio); accurate representation of the default theme look; PNG format preferred.               |
| **[[Licensing & Copyright]]**               | Legal compliance.                                                               | GPLv2 or later; copyright declaration in `style.css` matching the author.                                      |
| **[[Privacy]]**                             | Data handling.                                                                  | No external resource loading without user consent (e.g., Google Fonts must be locally hosted or consented).    |
