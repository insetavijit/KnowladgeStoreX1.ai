|**[[Xdebug & Step Debugging]]**|Step through code to inspect execution flow.|Breakpoints; IDE integration (VS Code); call stack; variable watch; remote debugging.|Xdebug enables interactive, line-by-line PHP debugging.|

|**[[Plugin & Theme Debuggers]]**|Use WordPress-specific debugging plugins.|Query Monitor; Debug Bar; Health Check; Log Viewer; profiling hooks.|Debug plugins provide WP-aware diagnostics.|

|**[[Performance Debugging]]**|Identify bottlenecks beyond functional errors.|PHP profilers; slow query logs; object cache stats; Lighthouse.|Performance debugging targets speed, not just correctness.|
|**[[Reproducibility & Isolation]]**|Isolate bugs to minimal failing cases.|Plugin/theme deactivation; Health Check mode; clean installs; binary search.|Isolating variables leads to faster fixes.|
|**[[Common Debugging Pitfalls]]**|Avoid practices that create new problems.|Leaving debug on in prod; ignoring logs; guessing fixes; not versioning changes.|Poor debugging hygiene causes security and stability risks.|
