# 1.4.5 Table Generation

| **Section** | **Focus** | **Key Utilities / Concepts** | **One-Line Recall** |
| :--- | :--- | :--- | :--- |
| **[[1.4.5.1 Markdown Tables]]** | Visual grids | `| Col1 | Col2 |`, alignment syntax, human-readable | Markdown tables are the best way to present comparative data in a chat interface. |
| **[[1.4.5.2 CSV Output]]** | Data export | Comma-separated values, header rows, special char quoting | Generating valid CSV allows the user to copy-paste the output directly into Excel or Google Sheets. |
| **[[1.4.5.3 Column Alignment]]** | Formatting | `|:---|:---:|---:|`, whitespace padding | Aligning text (left/center/right) improves legibility, especially for numerical columns. |
| **[[1.4.5.4 Handling Empty Cells]]** | Sparsity | Null handling, placeholders (`-`), preserving alignment | Explicitly handling missing data prevents the table columns from becoming misaligned. |
| **[[1.4.5.5 Header Definitions]]** | Semantics | Clear labels, unit definitions (`Price ($)`), sorting criteria | Unambiguous headers are crucial for the user to interpret the data correctly. |
| **[[1.4.5.6 Multi-Line Content in Cells]]** | Formatting issues | `<br>` tags, removing newlines, truncation | Markdown tables break if a cell contains a raw newline; replacing them with `<br>` or spaces is required. |
| **[[1.4.5.7 Data Transposition]]** | Orientation | Row-oriented vs Column-oriented, pivoting | Sometimes presenting data as a list of key-values is better than a wide table; know when to pivot. |
| **[[1.4.5.8 Spreadsheet Formulas]]** | Active data | `=SUM(A1:A5)`, generating formulas, dynamic cells | LLMs can generate spreadsheets that include working formulas, not just static values. |
| **[[1.4.5.9 Comparison Matrices]]** | Decision support | Feature vs Product grid, checkmarks, scoring | A comparison matrix is a specific table type optimized for helping users make decisions. |
| **[[1.4.5.10 Visual Table Formatting]]** | Styling | Bold headers, highlighting rows, infinite scroll consideration | Formatting choices should respect the limitations of the chat UI (e.g., avoid ultra-wide tables). |

This table represents a **complete guide to Table Generation**, turning text into organized grids.
