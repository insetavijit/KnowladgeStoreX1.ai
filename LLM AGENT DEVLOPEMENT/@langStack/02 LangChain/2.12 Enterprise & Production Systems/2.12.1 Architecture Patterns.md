| **[[2.12.1 Architecture Patterns]]** | Blueprint | high-level structural patterns for building LLM apps | The construction plans for the skyscraper. |
| **[[Microservices]]** | Decoupling | separating the "Researcher" agent from the "Writer" agent into different services | "Don't put all your eggs in one monolithic basket." |
| **[[Serverless]]** | Scaling | running individual chain steps on AWS Lambda or Google Cloud Functions | "Pay only when the code runs." |
| **[[Event-Driven]]** | Async | using message queues (Kafka/RabbitMQ) to trigger agents | "When a new file lands in S3, wake up the ingestion agent." |
| **[[Domain-Driven Design]]** | Logic | organizing agents around business domains (Sales, Support, HR) | Structuring the code like the org chart. |
| **[[Clean Architecture]]** | Maintenance | keeping business logic separate from framework code (LangChain) | "If we switch from LangChain to LlamaIndex, the business logic stays same." |
| **[[Scalability]]** | Growth | designing systems that can handle 1 user or 1 million users | "It works on my laptop, but will it work on Black Friday?" |
