| **[[4.6.5 Parsing Algorithms Overview]]**| Computational methods to build parse trees                     | complexity (O(n^3) vs O(n)), deterministic vs probabilistic, top-down vs bottom-up         | The code that turns a flat list of words into a structured tree.       |
| **[[Shift-Reduce Parsing]]**            | Fast, linear-time transition-based parsing                     | stack, buffer, actions (SHIFT, LEFT-ARC, RIGHT-ARC), greedy decoding, arc-standard         | Processing words left-to-right like a stack calculator.                |
| **[[CYK Algorithm]]**                   | Bottom-up chart parsing for CFGs                               | dynamic programming, O(n^3) complexity, requires Chomsky Normal Form (CNF)                 | Building the pyramid of the sentence from the bottom up guarantees optimal trees.|
| **[[Earley Parser]]**                   | Top-down chart parsing                                         | handles any CFG, prediction/scanning/completion steps, O(n^3) worst case                   | Guessing the structure and checking if the words fit.                  |
| **[[Graph-Based Parsing]]**             | Scoring entire trees at once                                   | Maximum Spanning Tree (MST), Chu-Liu-Edmonds algo, global scoring, higher accuracy         | Finding the strongest set of connections in the web of all possible arrows.|
| **[[Neural Parsing]]**                  | Deep learning approaches                                       | BiAffine attention (Dozat & Manning), pointer networks, end-to-end tree prediction         | Letting a neural network predict the arcs and labels directly.         |
